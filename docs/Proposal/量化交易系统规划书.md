# 东方财富掘金实盘执行系统规划书（第一期）

## 文档定位

本文只定义第一期最小闭环，不展开第二期。第一期目标是在东方财富掘金仿真账户中跑通“多标的持仓识别 -> 规则判断 -> 自动卖出 -> 回报收口 -> 日志审计”。

## 一、目标与边界

### 1.1 目标

- 连接东方财富掘金仿真账户
- 读取账户资金、全部持仓和相关行情
- 自动识别全部可卖持仓
- 按固定止盈止损规则触发自动卖出
- 接收委托与成交回报
- 防止同一标的重复卖出
- 留存可审计日志

### 1.2 范围

- 账户、持仓、行情、委托、回报全部统一走东方财富掘金官方接口
- 开发语言固定为 Python 3.10+
- 运行形态为单进程常驻策略程序
- 只做自动卖出，不做自动买入
- 持仓默认由人工先建，系统不负责开仓
- 第一版按单账户、单策略、统一参数运行

### 1.3 非范围

- 前端
- 回测
- 数据库主链路
- 对外 Web API
- 多账户
- 多策略组合
- 动态脚本加载
- 第二期和远期路线

### 1.4 成功标准

在东方财富掘金仿真环境中，系统能够自动识别账户当前全部可卖持仓，并对满足固定止盈止损条件的标的执行自动卖出；整个过程中，每个标的都具备独立状态管理、重复卖单防护和可审计日志。

## 二、技术基线

| 维度 | 第一阶段方案 |
| --- | --- |
| 交易环境 | 东方财富掘金仿真账户 |
| 接口来源 | 东方财富掘金官方接口 |
| 开发语言 | Python 3.10+ |
| 运行方式 | 单进程常驻策略程序 |
| 策略范围 | 固定止盈止损 |
| 交易方向 | 仅自动卖出 |
| 观测手段 | 本地日志 + 掘金委托/成交回报 |
| 部署目标 | 先跑通最小闭环，不追求平台化 |

## 三、系统架构

### 3.1 四层架构

| 分层 | 职责 |
| --- | --- |
| 基础设施层 | 配置、启动停止、调度、交易时段控制、日志、异常处理 |
| 数据接入层 | 掘金账户连接、资金读取、持仓读取、行情读取、委托调用、回报接收 |
| 核心决策层 | 逐标的状态管理、止盈止损判断、卖出许可判断 |
| 交易执行层 | 卖单执行、委托跟踪、成交收口、防重复卖单 |

### 3.2 架构约束

- 所有外部交易与行情调用统一经过数据接入层
- 核心决策层只负责“该不该卖”，不直接发单
- 交易执行层只负责“怎么卖”和“卖后状态怎么收口”
- 状态必须按标的隔离，禁止用单一全局布尔值代替
- 测试与质量保障单独成章，不并入任何一层偷写一句话

## 四、核心运行链路

1. 程序启动，加载配置并初始化日志。
2. 基础设施层判断当前运行环境和交易时段。
3. 数据接入层连接掘金仿真账户并确认账户可用。
4. 数据接入层读取账户资金和当前全部持仓。
5. 核心决策层筛选全部可卖持仓，并为每个标的建立或更新状态。
6. 数据接入层获取相关标的最新行情。
7. 核心决策层依据持仓成本、当前价格、止盈比例和止损比例判断是否触发卖出。
8. 核心决策层检查该标的是否已有未完成卖单，以及当前是否允许发单。
9. 交易执行层发起卖单，并记录触发原因、数量、价格和委托结果。
10. 数据接入层接收委托与成交回报，交易执行层更新逐标的执行状态。
11. 程序继续轮询全部持仓，直到停止。

## 五、测试与质量保障

### 5.1 测试范围

- 单元测试：止盈判断、止损判断、交易时段判断、防重复卖单、逐标的状态隔离
- 集成测试：持仓读取到决策输出、决策输出到交易执行、委托回报到状态收口
- 仿真冒烟：账户连通、持仓读取、触发卖出、回报记录、异常落日志

### 5.2 最低质量门槛

- 无持仓时不发单
- 非交易时段不发单
- 同一标的不重复发单
- 接口失败和回报异常必须落日志
- 核心判断口径可通过测试复现

## 六、里程碑

| 里程碑 | 目标 | 完成定义 |
| --- | --- | --- |
| M0 | 环境与账户连通 | 能完成鉴权，并读取账户资金、全部持仓和持仓标的行情 |
| M1 | 数据接入跑通 | 账户、持仓、行情、委托、回报链路可稳定调用 |
| M2 | 核心决策与状态管理 | 多标的状态管理、止盈止损判断、卖出许可判断可运行 |
| M3 | 自动卖出执行闭环 | 满足条件的标的能自动卖出，并完成委托与成交状态收口 |
| M4 | 测试、日志与稳定运行 | 关键测试可执行，日志齐全，程序可连续稳定运行 |

## 七、风险与运行约束

- 仿真环境只能验证链路正确性，不等于真实成交结果
- 非交易时段允许启动和读取状态，但禁止发起新卖单
- 同一标的若已有未完成卖单，必须禁止再次主动发单
- 多个标的必须独立维护状态，不能互相覆盖
- 行情、账户、委托、回报异常必须进入明确状态并落日志
- 账户无持仓时系统只能空转，不允许自行开仓

## 八、结论

第一期方案只追求一件事：把东方财富掘金仿真环境下的多标的自动卖出闭环跑通。只要 `M0 -> M4` 能按上述口径落地，这一版规划书就达到了项目基线要求。
