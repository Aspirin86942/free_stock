# 核心决策层 Spec

## 1. 文档目标

本文定义第一期实盘执行系统的核心决策层边界、状态模型、判断规则、输出契约和验收标准。

核心决策层只回答一个问题：当前这个标的现在能不能卖。它不直接调用掘金接口，也不直接承担委托提交流程。

## 2. 本层定位

核心决策层统一负责：

- 逐标的状态管理
- 可卖持仓筛选
- 固定止盈止损判断
- 卖出许可判断
- 卖出信号输出

本层是系统的“业务判断中心”，但不是“接口调用中心”。

## 3. 范围与非范围

### 3.1 本层范围

- 为每个可卖持仓标的建立独立运行状态
- 根据持仓成本、当前价格和阈值判断是否触发
- 判断当前是否处于允许发单的状态
- 判断当前是否已有未完成委托
- 生成卖出信号和触发原因

### 3.2 不在本层范围

- 配置加载和轮询调度
- 行情和持仓的原始读取
- 委托提交
- 委托和成交回报适配
- 外部接口异常重试

## 4. 与里程碑关系

核心决策层直接支撑以下里程碑：

- `M2 核心决策与状态管理`
- `M3 自动卖出执行闭环`

`M2` 的关键不在于“已经能卖出”，而在于：系统已经能稳定地对多标的独立判断，并输出明确的卖出许可结论。

## 5. 输入、输出与依赖

### 5.1 输入

- 持仓快照列表
- 行情快照列表
- 逐标的现有状态
- 当前交易时段状态
- 止盈比例和止损比例
- 未完成委托信息

### 5.2 输出

- 逐标的决策结果
- 是否触发卖出
- 触发原因
- 当前状态更新建议
- 是否允许进入交易执行层

### 5.3 依赖

- 基础设施层提供的交易时段判断
- 数据接入层提供的持仓、行情和委托信息
- 交易执行层回写的执行状态结果

## 6. 状态模型

第一期必须按标的维护独立状态，建议最小状态如下：

| 状态 | 含义 |
| --- | --- |
| `idle` | 有持仓，但本轮未触发 |
| `triggered` | 已满足条件，待进入执行层 |
| `submitting` | 正在提交卖单 |
| `submitted` | 卖单已发出，等待回报 |
| `partially_filled` | 部分成交 |
| `filled` | 已全部成交 |
| `cancelled` | 已撤单 |
| `failed` | 委托失败或状态异常 |

约束如下：

- 状态粒度必须是 `symbol`
- 一个标的的状态变化不能影响其他标的
- 程序重启后应至少能根据持仓和未完成委托恢复基本状态

## 7. 判断规则

### 7.1 可卖持仓筛选

- 只处理 `available_volume > 0` 的持仓
- 无持仓或可卖数量为零时，直接输出不触发

### 7.2 止盈规则

- 当 `current_price >= cost_price * (1 + take_profit_ratio)` 时，触发止盈卖出

### 7.3 止损规则

- 当 `current_price <= cost_price * (1 - stop_loss_ratio)` 时，触发止损卖出

### 7.4 卖出许可判断

同时满足以下条件才允许进入执行层：

- 当前处于允许发单的交易时段
- 标的存在可卖数量
- 当前没有未完成卖单
- 当前状态不是 `submitted`、`partially_filled` 或 `filled`

## 8. 决策输出契约

每个标的至少输出以下字段：

| 字段 | 说明 |
| --- | --- |
| `symbol` | 标的代码 |
| `triggered` | 是否触发 |
| `trigger_type` | `take_profit` 或 `stop_loss` |
| `current_price` | 当前价格 |
| `cost_price` | 持仓成本 |
| `threshold_price` | 当前触发阈值 |
| `available_volume` | 可卖数量 |
| `can_execute` | 是否允许进入执行层 |
| `reason` | 触发或阻断原因 |
| `next_state` | 建议状态 |

如果未触发，也必须给出明确原因，例如：

- `no_position`
- `no_sellable_volume`
- `not_in_trading_session`
- `existing_open_order`
- `price_not_reached`

## 9. 异常处理策略

第一期禁止把判断异常当作“未触发”静默吞掉。

处理口径如下：

- 持仓缺失关键字段：当前标的进入错误路径并落日志
- 行情缺失关键字段：当前标的不进入判断，并落日志
- 阈值参数非法：直接阻止系统启动
- 状态不一致：标记为 `failed` 或明确异常状态，并等待人工介入

## 10. 日志与审计要求

本层至少记录以下信息：

- 标的代码
- 当前价格
- 持仓成本
- 可卖数量
- 止盈阈值
- 止损阈值
- 决策结果
- 阻断原因或触发原因
- 状态变化前后结果

## 11. 测试要求

本层至少覆盖以下测试：

- 止盈触发正确
- 止损触发正确
- 未到阈值不触发
- 非交易时段不允许进入执行层
- 已有未完成卖单不允许进入执行层
- 多标的状态互不污染
- 程序重启后的基础状态恢复逻辑可复现

## 12. 本层完成定义

核心决策层可视为完成，至少需要满足以下条件：

- 能识别全部可卖持仓
- 能按标的维护独立状态
- 能正确完成止盈止损判断
- 能正确阻断非交易时段和重复卖单
- 能输出统一卖出信号和阻断原因
