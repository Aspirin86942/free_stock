# 量化交易系统规划书 Markdown 标准化与数据源改版设计

## 1. 背景

当前文档 [量化交易系统规划书.md](D:\Program_python\free_stock\Proposal\量化交易系统规划书.md) 虽然使用了 `.md` 扩展名，但实际混用了 Obsidian callout、伪表格和裸 `mermaid` 文本，阅读体验更接近笔记体 Markdown，而不是仓库内可直接渲染的标准 GitHub Markdown。

本次改版目标有两个：

- 把全文整理为标准 GitHub Markdown，保证在仓库、编辑器和代码托管平台里能稳定阅读。
- 把数据采集结论改成已确认的首期方案，并同步修正前后文中与该结论冲突的表述。

## 2. 已确认结论

### 2.1 首期范围

- 首期支持范围固定为 `A 股 + ETF`。
- 期货从首期规划中移除，不再出现在首期系统目标、里程碑和边界摘要里。

### 2.2 数据采集结论

`1.6.1` 改为：

`数据采集：新浪行情适配器 + AkShare + 网络爬虫`

数据源职责固定如下：

- 新浪行情适配器：分钟级历史与近实时行情主来源，优先覆盖 A 股、ETF、指数。
- AkShare：补数、基础资料、批量快照和备用来源，不作为唯一主源。
- 网络爬虫：公告、新闻、舆情和缺失字段补充，不承担核心行情主链路。

以下方案从首期基线中移除：

- 腾讯 API 主数据源表述。
- BaoStock 主或辅数据源表述。

### 2.3 分钟级数据边界

- 首期研究与回测链路的刚需是分钟级历史数据。
- 高频 Tick、逐笔成交、Level-2 数据不属于首期前提。
- 分钟级近实时行情属于模拟盘与准实盘阶段的重要能力，但不作为首期研究型系统落地的阻塞条件。

## 3. Markdown 标准化范围

本次只做“标准化整理 + 已确认结论修正”，不做无关内容重写。

保留项：

- 原有章节编号和章节顺序。
- 文档开头的 YAML front matter。
- 架构图内容本身。

调整项：

- 把 `[!abstract]`、`[!important]`、`[!tip]`、`[!note]`、`[!summary]` 改为标准标题或引用块。
- 把当前文本型伪表格改为标准 Markdown 表格。
- 把裸 `flowchart TB` 内容改为 fenced code block：` ```mermaid `。
- 统一标题层级、列表缩进、空行和段落分隔。

不做项：

- 不在本轮新增新模块。
- 不在本轮扩写实现细节。
- 不在本轮引入新的数据源候选讨论。

## 5. 数据源结论的依据

截至 `2026-04-07`，已完成以下核验：

- 新浪历史 K 线接口实测可返回 `1/5/15/30/60/240` 分钟数据。
- 新浪实时快照、分时和逐笔接口实测可返回 A 股、ETF、指数样例数据。
- `AkShare 1.17.32` 在 `conda` 的 `test` 环境可正常返回：
  - A 股 1 分钟历史数据
  - ETF 1 分钟历史数据
  - A 股实时快照
  - ETF 实时快照

因此，首期文档中把“新浪行情适配器 + AkShare + 网络爬虫”作为结论是有验证支撑的；但由于新浪路线本质上应按“适配器接入”而不是“官方稳定 SLA 数据服务”来描述，所以文档中必须使用“适配器”口径，避免事实表述过度。

- 首期范围统一为 `A 股 + ETF`，不再残留期货首期表述。
