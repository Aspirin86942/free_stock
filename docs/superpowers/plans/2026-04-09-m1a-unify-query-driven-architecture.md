# 查询驱动架构统一实施记录

## Goal

将项目从混合式历史口径统一收敛为当前查询驱动口径，并同步清理代码、测试和文档中的旧设计残留。

## 统一后的主路径

```text
提交结果 -> 查单 -> 查成交 -> 内部事件 -> 状态聚合/状态机
```

## 影响范围

- 当前生效代码
- 单元测试与集成测试
- 独立烟雾脚本
- 规划书
- 历史 spec 与计划文档

## 实施项

1. 删除旧混合链路下的多余处理器与测试残留
2. 统一 `GMTradeQueryGateway` 只保留提交、查单、查成交职责
3. 将根目录 `test.py` 收敛为 `scripts/query_smoke_test.py` 纯查询烟雾脚本
4. 将 M1 服务统一为“查询结果 -> 内部事件 -> 聚合报告”
5. 将历史文档整体修订到新口径

## 验证命令

```powershell
conda run -n stock_analysis pytest
```

## 完成标准

- 当前代码主路径只保留查询驱动
- 测试断言与文件命名不再表达旧方案
- 规划书、spec、plan 与实际代码口径一致
- 旧术语残留搜索结果为空
